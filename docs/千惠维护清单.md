## 四、同步注意事项

### 4.1 本地 → 云端（代码 / 配置）

**关键：云端配置有 4 个差异项不能被本地覆盖**

| 文件 | 配置项 | 本地值 | 云端必须保持的值 | 原因 |
|------|--------|--------|-----------------|------|
| `config/bot_config.toml` | `webui.host` | `127.0.0.1` | `0.0.0.0` | 允许远程访问 |
| `config/bot_config.toml` | `webui.allowed_ips` | `127.0.0.1` | `0.0.0.0` | 放开 IP 白名单 |
| `config/bot_config.toml` | `ws_server_host` | `127.0.0.1` | `0.0.0.0` | Docker 容器通信 |
| `config/bot_config.toml` | `enable_tts` | `true` | `false` | 云端无 SoVITS |

**推荐做法**：同步 `bot_config.toml` 前先 `diff` 一下：

```powershell
# 对比本地和云端配置
ssh ubuntu@82.156.88.63 "cat /opt/maibot/config/bot_config.toml" | diff - .\config\bot_config.toml
```

如果本地改了这 4 项中的任何一个，传完后要在云端改回云端值。

### 4.2 适配器配置（NapCat 适配器有自己的过滤层）

文件：`plugins/MaiBot-Napcat-Adapter/config.toml`

关键配置（云端当前值）：

```toml
[chat]
enable_chat_list_filter = false    # 关闭白名单过滤，所有人可私聊
ban_qq_bot = true                  # 屏蔽群里的 QQ 官方机器人
```

⚠ 同步时不要把本地的适配器配置覆盖到云端，除非你确认两边一致。

### 4.3 云端 → 本地（数据备份）

只做备份，不回写。云端是唯一数据源，本地备份仅供灾难恢复。

**不要把本地 `data/` 推到云端**——会覆盖云端最新的记忆和聊天记录。

### 4.4 数据库路径差异

云端数据库里 `images.full_path` 已从 Windows 路径修复为 Linux 路径（`/opt/maibot/data/emoji/...`）。如果从本地同步新的 `data/MaiBot.db` 到云端，需要重新跑路径修复脚本：

```bash
cd /opt/maibot
python3 << 'PYEOF'
import sqlite3
from pathlib import Path
conn = sqlite3.connect("data/MaiBot.db")
cursor = conn.execute("SELECT id, full_path, image_type FROM images")
updates = []
for row in cursor:
    record_id, full_path, image_type = row
    if not full_path or "\\" not in full_path:
        continue
    filename = Path(full_path).name
    if image_type == "emoji":
        new_path = str(Path("/opt/maibot/data/emoji") / filename)
    elif image_type == "image":
        new_path = str(Path("/opt/maibot/data/images") / filename)
    else:
        continue
    updates.append((new_path, record_id))
if updates:
    conn.executemany("UPDATE images SET full_path = ? WHERE id = ?", updates)
    conn.commit()
    print(f"修复完成: {len(updates)} 条记录")
conn.close()
PYEOF
```

## 六、NapCat 定时重启（已配置）

云端已配置 crontab，每天凌晨 4:00 自动重启 NapCat 容器，释放内存、刷新连接：

```bash
# 查看当前 crontab
crontab -l

# 如果没有，手动添加
(crontab -l 2>/dev/null; echo "0 4 * * * cd /opt/maibot && docker compose restart napcat") | crontab -
```
